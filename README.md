# VNL Tournament Simulator 🏐

An AI-powered volleyball tournament simulator built on the VNL (Volleyball Nations League) dataset. The system simulates a full tournament with group stages and knockout rounds, explains each match result using SHAP explainability, and generates natural language match analyses using a HuggingFace language model.

---

## What it does

Select 16 national teams, simulate a full VNL-style tournament, and get set-by-set results for every match — from group stage through the final. For any match, you can request an AI-generated analysis that explains which statistics drove the result, powered by SHAP values and a HuggingFace language model.

---

## How it works

**Data aggregation** — individual player statistics are aggregated per team. Three efficiency metrics are derived: Attack Efficiency, Serve Efficiency, and Block Efficiency.

**Team Strength Score** — all statistics are normalized between 0 and 1 using MinMaxScaler. Positive stats (points, efficiency) contribute positively; errors reduce the score. Final scores are scaled to 0-100.

**Match simulation** — win probability is calculated using a logistic function on the difference between two team strength scores. Sets are simulated individually in best-of-5 format with realistic point ranges.

**Tournament format** — group stage with 4 groups of 4 teams (round-robin, 3 points per win), followed by quarterfinals, semifinals, third place match, and final.

**SHAP explainability** — since the dataset has no historical match results, synthetic match data is generated by simulating all team pair matchups. An XGBoost classifier is trained on stat differences between teams. SHAP TreeExplainer identifies which statistics contributed most to each prediction.

**Natural language analysis** — the top 3 SHAP features and match result are passed as a structured chat prompt to SmolLM2-1.7B-Instruct. The model generates a 3-4 sentence match analysis. The model is loaded via AutoModelForCausalLM rather than the pipeline abstraction — the pipeline was applying a hardcoded max_length=20 from the model's generation_config, which prevented any text from being generated.

## Dataset

**Source:** [VNL 2025 Player Data — Kaggle](https://www.kaggle.com/datasets/joshuali12/vnl-2025-player-data)

305 players across 18 national teams with 19 statistical columns covering attack, block, serve, set, dig, and receive performance.

## Tech Stack

| Component | Technology |
|---|---|
| Data processing | pandas |
| Normalization | scikit-learn (MinMaxScaler) |
| Prediction model | XGBoost |
| Explainability | SHAP (TreeExplainer) |
| Natural language generation | HuggingFace Transformers (SmolLM2-1.7B-Instruct) |
| UI | Streamlit |
